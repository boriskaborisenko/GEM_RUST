# Квантовый Контекс-Манифест Проекта — GEM_RUST

Этот документ является **главным архитектурно-математическим справочником** проекта `GEM_RUST`. Он разработан специально для мгновенной загрузки контекста (ИИ-ассистентами и инженерами) и содержит исчерпывающее описание структуры, логики и формул торгового ядра.

---

## 1. Архитектурная карта модулей

Ядро робота спроектировано на реактивной, асинхронной и многопоточной архитектуре на базе **Rust Tokio** и разбито на следующие изолированные модули:

```text
GEM_RUST/src/
├── main.rs              # Точка входа. Управление AppState, mpsc event-loop, promotion окон и дашборд.
├── client.rs            # Вебсокет-клиенты: Chainlink (спот-цены) и Polymarket CLOB (биды/аски YES/NO).
├── trader.rs            # Бумажный трейдер (PaperTrader). Ведение Portfolio, WindowState, сделок и авто-логирование в CSV.
├── config.rs            # Парсер конфигурации config.json на базе serde.
├── volatility.rs        # Bybit WebSocket коннектор. Сбор 1m свечей и RMA расчет скользящего ATR(14) в реальном времени.
├── analytics.rs         # Офлайн ИИ-Аудитор. OAuth2 авторизация (gcp_auth), Vertex AI и интеграция с Gemini 2.5 Pro.
└── strategy/            # Плагин-движок стратегий (stratBox)
    ├── mod.rs           # Диспетчер плагинов. Определение трейта TradeStrategy и роутер StrategyEngine.
    ├── strategy_a.rs    # Стратегия А (Simple Both): симметричный закуп, фиксированный тейк.
    ├── strategy_b.rs    # Стратегия Б (Asymmetric Ladder): асимметрия, лесенка, усыхание порогов Theta Decay.
    ├── strategy_c.rs    # Стратегия Ц (Dynamic Break-Even): первая продажа по цели, вторая по формуле безубытка.
    └── strategy_d.rs    # Стратегия Д (Dynamic Grid): 3-ступенчатый Grid, Dynamic BUY, Time-Decay Crossover Filter.
```

## 2. Паспорт 4-х Торговых Стратегий движка `stratBox`

Движок стратегий полностью полиморфен и унаследован от единого интерфейса `TradeStrategy`:

### Стратегия А: «Simple Both» (Базовая «тупая»)
* **Название в конфиге:** `"simple_both"`
* **Вход:** За 1 минуту до старта покупает UP и DOWN строго поровну по цене `0.50 / 0.51$` (или наоборот).
* **Выход:** Как только бид UP или DOWN коснулся `>= 0.65$`, бот продает 100% стороны. Стороны выходят независимо.

### Стратегия Б: «Asymmetric Ladder» (Асимметричная лесенка)
* **Название в конфиге:** `"asymmetric_ladder"`
* **Вход:** Асимметричный закуп (55% на UP сильную сторону, 45% на DOWN слабую сторону).
* **Выход:** Каждая сторона распродается лесенкой — порциями по **`50%`** объема.
* **Временной распад (Theta Decay):** Если `"decayEnabled": true`, по мере старения опциона цели лесенки умножаются на понижающий коэффициент:
  * `time_pct < 50.0%` (0–7.5 мин): коэффициент **`1.0`** (цели `0.62/0.72` и `0.70/0.85`).
  * `50.0% <= time_pct < 80.0%` (7.5–12 мин): коэффициент **`0.90`** (цели усыхают на 10%).
  * `time_pct >= 80.0%` (12+ мин): коэффициент **`0.80`** (цели усыхают на 20% для быстрого сброса перед финалом).

### Стратегия Ц: «Dynamic Break-Even» (Динамический безубыток)
* **Название в конфиге:** `"dynamic_breakeven"`
* **Вход:** Асимметричный закуп (например, 55/45).
* **Выход 1:** Первая добежавшая до цели нога продается на 100% по жесткому тейку `0.65$`. Нам возвращается кэш `Cash_Returned`.
* **Выход 2:** Бот ждет разворота спота за страйк. На секунду пересечения бот считывает Bid второй стороны и сравнивает его с динамическим безубытком по формуле:
  $$\text{Min\_Safe\_Price} = \frac{\text{Spent} - \text{Cash\_Returned}}{\text{Remaining\_Shares}} + \text{slippageBuffer}$$
  Если `Bid >= Min_Safe_Price` — продаем остатки в плюс/ноль. Если нет (распад слишком сильный) — бот НЕ продает в минус, а оставляет позицию жить дальше (выжидаем глубокий разворот, либо отдаем под экспирацию).

### Стратегия Д: «Dynamic Grid» (3-ступенчатый Grid + Dynamic BUY + Time-Decay Filter)
Наш самый мощный, отказоустойчивый высокочастотный алгоритм.
* **Название в конфиге:** `"dynamic_grid"`
* **Вход:** Асимметричный закуп (55/45). Сильная UP (`16.5$`), слабая DOWN (`13.5$`).
* **Сетка фиксации (UP):** Слив сильной стороны по 3-м ступеням (40% по `0.58$`, 40% по `0.66$`, 20% по `0.75$`).
* **Усреднение Dynamic BUY (DOWN):** Если сильная UP коснулась Ступени 3 (`0.75$`), а время раунда `time_pct <= 60%`, бот моментально докупает слабую DOWN по лимитной цене **`0.15$`** объемом 50% от первоначального, сбивая среднюю цену входа с `0.50$` до сладких **`0.38$`**!
* **Временной фильтр процентов на пересечении:**
  * `time_pct < 50.0%`: моментальная рыночная продажа слабой ноги для фиксации Win (бид `>= 0.43$`).
  * `50.0% <= time_pct <= 75.0%`: выход только при условии полной окупаемости раунда по формуле `Realized_PnL >= 0`.
  * `time_pct > 75.0%`: полная блокировка продаж на страйке, даем слабой стороне долететь до конца в надежде на чистый полный разворот.

### 🚀 Адаптивная Модель Управления Капиталом (Реализована!):
* **Интеллектуальный Динамический Сайзинг (Dynamic Sizing):** Бюджет на раунд рассчитывается динамически на базе текущего показателя **Equity (Эквити)** портфеля по формуле:
  $$\text{Бюджет}_{\text{целевой}} = \text{Equity} \times \left(\frac{\text{windowBudgetPct}}{100}\right)$$
  Полученный бюджет строго ограничивается защитными порогами `minWindowBudget` и `maxWindowBudget` из файла `config.json`.
* **Умный сплит через стратегию:** При входе в сделку бот автоматически анализирует цены Ask, но финальный split теперь задает активная стратегия через `EntrySignal`. Для Strategy D split зависит от ATR-режима, а не от глобального параметра в `config.json`.
* **Динамические pre-start ask пороги:** Все хардкод-значения удалены из стратегий. Каждая стратегия полностью динамически считывает коридор входа (`minSideAsk` / `maxSideAsk`) из `config.json` (дефолтное значение сужено до `0.48 - 0.52`).
* **Встроенный расчет процентного отклонения от страйка (PTB):** Метод `market.get_ptb_deviation(spot_price)` в `src/client.rs` рассчитывает отклонение в валюте и в процентах со сверхвысокой точностью. Это значение выводится на дашборд (`Dist`) и доступно в любой стратегии в качестве триггеров.

## 3. Менеджер волатильности (Bybit WS ATR Engine)

Модуль **`src/volatility.rs`** запускается в фоне при старте приложения:
* Подключается к Bybit WebSocket Public API (`wss://stream.bybit.com/v5/public/linear`).
* Подписывается на поток минутных свечей: `kline.1.BTCUSDT`.
* Каждую минуту собирает подтвержденные (`confirm: true`) бары в кольцевой буфер `history` объемом **15 свечей**.
* Вычисляет **True Range (TR)**: `Max(H-L, |H-C_prev|, |L-C_prev|)`.
* Сглаживает TR по классической скользящей средней Уайлдера (RMA) с периодом **14**.
* Отдает текущее значение ATR(14) в долларах BTC через метод `.get_current_atr()`.
* На дашборде выводится живое значение: `BTC ATR(1m): $182.40`.

## 4. Офлайн ИИ-Аналитик (Vertex AI & Gemini 2.5 Pro)

Бот полностью подготовлен к ИИ-самообучению и аудиту без влияния на задержки HFT-ядра:

1. **Авто-логирование в CSV (`src/trader.rs`):** При закрытии каждого опциона (метод `close_window`), если мы входили в него (`spent > 0.0`), бот дописывает строку в файл **`logs/trades_history.csv`** в формате:
   `window_id,slug,spent,returned,pnl,timestamp`
2. **Модуль аналитики (`src/analytics.rs`):**
   * Авторизуется в Google Cloud Platform по JSON-файлу сервисного аккаунта `credentials/service-account-key.json` через библиотеку `gcp_auth` (версия 0.12, используется `CustomServiceAccount` и `TokenProvider`).
   * Генерирует OAuth 2.0 Bearer Token.
   * Шлет HTTP POST запрос на Vertex AI к модели **gemini-2.5-pro**.
   * Модель считывает CSV-массив логов, находит скрытые паттерны убытков в неволатильные часы, оценивает эффективность Dynamic BUY и точек Time-Decay пересечений и выгружает готовые рекомендации по калибровке сетки в локальный файл `logs/gemini_strategy_report.txt`.

## 5. Управление, Сборка и Запуск

Все настройки вынесены в один файл конфигурации `GEM_RUST/config.json`.

### Команды консоли:
```bash
# Переход в директорию
cd GEM_RUST

# Сборка и проверка
cargo check
cargo build

# Запуск бота (BTC/ETH/SOL и интервалы 5m/15m)
cargo run -- BTC 5m
cargo run -- ETH 15m
cargo run -- SOL 5m
```

Этот проект — идеальный пример сверхбыстрого HFT-ядра на Rust, управляемого из динамического JSON-конфига и снабженного ИИ-аудитом высокого класса!
