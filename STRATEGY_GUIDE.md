# Справочник Стратегии «Simple Cross» — Бот GEM_RUST

Этот файл содержит подробную и понятную инструкцию по устройству текущей торговой стратегии робота `GEM_RUST`. Здесь описано, по каким правилам бот входит и выходит из сделок, а также где и какие параметры менять для её точной корректировки.

---

## 1. Суть и Логика Стратегии

Стратегия **Simple Cross** — это простая, надежная, полностью реактивная (Event-Driven) стратегия, основанная на волатильности курсов криптоактивов в краткосрочных (5-минутных или 15-минутных) опционах Polymarket.

### Фаза 1. Вход в сделку (Pre-Start Entry)
За несколько минут до старта нового временного окна (интервала) бот подписывается на котировки через стаканы CLOB WebSocket.
* **Правило покупки:** Бот ждет момента, когда YES-контракт на рост (`UP`) и YES-контракт на падение (`DOWN`) будут торговаться по цене спроса (Ask) **ровно по 0.50$ / 0.51$** (или `0.51$ / 0.50$`).
* **Результат:** Бот закупает оба контракта, тратя суммарно ровно **1.01$ на пару контрактов**. Позиция полностью захеджирована, риск зафиксирован.

### Фаза 2. Выход из сделки (Live Exit)
Как только интервал начинается (окно переходит в статус `CURRENT` / `LIVE`), бот переходит на реактивный мониторинг цен лучших предложений покупателей (Bid) по сокетам CLOB WebSocket:
* **Продажа любой из сторон (Цель):** Бот непрерывно сравнивает Bid каждого из контрактов с целевым значением (по умолчанию **`0.65$`**).
* **Срабатывание цели:** Как только бид любого контракта уходит `>= 0.65$`, бот мгновенно продает его по текущей рыночной цене. Бот продолжает держать вторую сторону.
* **Продажа второй стороны:** Нам не важны никакие пересечения графиков спота или страйки! Мы просто тупо ждем, когда бид второго оставшегося контракта также вырастет до **`>= 0.65$`**, и сливаем его при первой возможности.
* **Профит:** При наличии волатильности мы продаем обе стороны по 0.65$ за штуку. Затратили **1.01$**, получили обратно **1.30$** (чистая прибыль **~30%**).

## 2. Где и как корректировать параметры?

### А. Настройки через файл конфигурации (`config.json`)
Самый простой способ изменить параметры робота без перекомпиляции кода — отредактировать файл `config.json` в корне папки `GEM_RUST`.

1. **Целевая цена продажи (exitBid):**
   * **Где менять:** В блоке `"sellStrategy"`.
   * **По умолчанию:** `0.65` (65 центов).
   * **Пример изменения на 70 центов:**
     ```json
     "sellStrategy": {
       "exitBid": 0.70
     }
     ```

2. **Сумма закупа одной пары контрактов:**
   * **Где менять:** Параметры `buyUpUsd` (объем закупа на рост) и `buyDownUsd` (объем закупа на падение) в блоке `"session"`.
   * **Пример изменения объема на 25$ за сторону:**
     ```json
     "session": {
       "startingBank": 100,
       "buyUpUsd": 25.0,
       "buyDownUsd": 25.0
     }
     ```

### Б. Корректировка параметров и правил в коде (требует `cargo build`)

Если вы хотите поменять математические правила или пороговые значения, которые зашиты в коде, вам понадобятся следующие файлы в папке `src/`:

1. **Экстренный стоп по времени (Emergency 15% Time Stop):**
   * **Что делает:** Если до конца опциона осталось менее 15% времени (45 секунд для 5м рынка), а цель в 0.65$ не достигнута, бот сливает остатки позиции, если за них дают хотя бы `>= 0.20$`, защищая баланс от сгорания в ноль.
   * **Где менять код:** В файле `src/strategy.rs`, внутри функции `process_live_tick`:
     * Порог времени (по умолчанию `0.15` / 15%):
       ```rust
       let emergency_time_threshold = (duration_sec * 0.15) as i64;
       ```
     * Порог минимальной цены экстренного спасения (по умолчанию `>= 0.20`):
       ```rust
       if win_state.up_shares > 0.0 && up_bid >= 0.20 && !state.up_sold
       ```

2. **Захват уровня страйка (Price to Beat / PTB) при старте опциона:**
   * **Что делает:** В момент наступления времени старта рынка бот автоматически и реактивно захватывает точный текущий спот-курс с Chainlink Spot WS в качестве PTB и сохраняет его.
   * **Где менять код:** В файле `src/main.rs`, в обработчике `MarketEvent::SpotTick`:
     ```rust
     if timestamp >= start.timestamp_millis() {
         curr.market.price_to_beat = Some(price);
         // ... сохранение страйка в портфель ...
     }
     ```

3. **Фильтр страйков при парсинге (`src/client.rs`):**
   * **Что делает:** Так как в описании рынка до его старта нет страйка, парсер `parse_strike_from_text` отсекает временной мусор из вопроса (минуты, даты) с помощью жестких ценовых порогов для каждого актива.
   * **Где менять код:** В файле `src/client.rs`, в функции `parse_strike_from_text`:
     ```rust
     let min_allowed = match asset_upper.as_str() {
         "BTC" => 10000.0, // BTC не может стоить дешевле 10k
         "ETH" => 1000.0,  // ETH не может стоить дешевле 1k
         "SOL" => 10.0,    // SOL не может стоить дешевле 10$
         _ => 0.0,
     };
     ```

## 3. Как запустить бота?

После любых изменений в файлах `.rs` (в папке `src/`) обязательно перекомпилируйте проект:
```bash
cargo build
```

Для запуска бота выполните команду (указав нужный актив и таймфрейм):
```bash
# Примеры запуска:
cargo run -- BTC 5m
cargo run -- ETH 15m
cargo run -- SOL 5m
```
