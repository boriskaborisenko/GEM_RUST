# Модель ИИ-Аудита Сделок и Оптимизации Стратегии (Gemini 2.5 Pro)

Этот документ содержит архитектурное ТЗ и руководство по проведению автономного ИИ-аудита торговых сессий робота `GEM_RUST` с помощью модели **Gemini 2.5 Pro** через Google Cloud Vertex AI.

---

## 1. Концепция: Автономный Самоанализ (Вариант Б)

Вместо опасного динамического вызова LLM во время быстрого HFT-интервала (риск задержек >1000мс), мы реализуем **пост-интервальный аудит**:

1. **Сбор данных:** При закрытии каждого торгового окна бумажного трейдера, модуль `src/trader.rs` автоматически записывает результаты сделки в файл логов **`GEM_RUST/logs/trades_history.csv`**.
2. **Анализ:** В конце торгового дня или недели, оператор или фоновый скрипт запускает модуль `analytics.rs`, который считывает накопленный CSV-файл и отправляет его в Vertex AI API.
3. **Вердикт:** Gemini 2.5 Pro анализирует корреляции между PnL, временем сделки, расширением спредов и волатильностью рынка (ATR) и генерирует структурированный отчет об оптимизации параметров и порогов сетки.

---

## 2. Структура Базы Данных Логов (`logs/trades_history.csv`)

Бот автоматически формирует и пополняет CSV-файл следующего формата:
```csv
window_id,slug,spent,returned,pnl,timestamp
1,btc-updown-15m-1779990300,30.00,22.11,-7.89,1779991200000
2,btc-updown-15m-1779992100,30.00,33.45,3.45,1779992100000
```

## 3. Системный Промпт для Gemini 2.5 Pro

Модель вызывается с жестко зашитым системным промптом (System Instruction) высокого уровня:

> *"Ты — ведущий квантовый аналитик и эксперт по HFT-стратегиям на крипторынке. Твоя задача — изучить логи торгового робота на Polymarket (15-минутные экспресс-рынки) и найти скрытые паттерны убытков. Выдели скрытые корреляции: время раунда, волатильность (ATR), поведение спреда и проскальзывание. Выдай конкретные рекомендации по оптимизации шагов сетки ордеров и тайм-фильтров. Твой ответ должен быть сугубо математическим, структурированным и основанным на цифрах из логов."*

## 4. Инструкция по Развертыванию Модуля Аналитики

### Шаг 1. Получение Ключа Сервисного Аккаунта Google Cloud
1. Перейдите в Google Cloud Console.
2. Создайте проект и включите в нем **Vertex AI API**.
3. Перейдите в раздел IAM & Admin -> Service Accounts.
4. Создайте сервисный аккаунт с ролью **Vertex AI User** (или `roles/aiplatform.user`).
5. Перейдите на вкладку Ключи (Keys), выберите "Add Key" -> "Create new key" -> **JSON**.
6. Скачайте JSON-файл и сохраните его в проект по пути:  
   **`GEM_RUST/credentials/service-account-key.json`**

### Шаг 2. Активация Зависимости в `Cargo.toml`
Библиотека аутентификации Google `gcp_auth` уже подключена в `Cargo.toml`:
```toml
gcp_auth = "0.12"
```

### Шаг 3. Использование Модуля в Коде
Для запуска еженедельного аудита вызовите менеджер аналитики из вашей точки входа:

```rust
use crate::analytics::VertexAnalytics;

#[tokio::main]
async fn run_audit() {
    // 1. Инициализируем аналитика из JSON-ключа
    let analytics = VertexAnalytics::new("credentials/service-account-key.json")
        .expect("Не удалось инициализировать Vertex AI");

    // 2. Считываем наши CSV-логи сделок
    if let Ok(logs) = std::fs::read_to_string("logs/trades_history.csv") {
        println!("Отправка накопленных логов в Gemini 2.5 Pro...");
        match analytics.analyze_trading_logs(&logs).await {
            Ok(report) => {
                println!("\n=== ОТЧЕТ ОБ ОПТИМИЗАЦИИ СТРАТЕГИИ ===");
                println!("{}", report);
                // Сохраняем отчет на диск
                std::fs::write("logs/gemini_strategy_report.txt", report).unwrap();
            }
            Err(e) => eprintln!("Ошибка ИИ-аудита логов: {:?}", e),
        }
    } else {
        println!("Файл логов logs/trades_history.csv пуст или еще не создан.");
    }
}
```

Этот подход обеспечивает умную, безопасную и полностью контролируемую ИИ-калибровку робота без создания задержек в реактивном торговом ядре на Rust!
