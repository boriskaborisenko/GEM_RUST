# Strategy J Critical Fix Checklist

Status: P0 implemented  
Scope: `GEM_RUST` Strategy J paper/runtime behavior  
Date: 2026-06-20

## Why This Exists

Strategy J has a strong target-exposure idea, but the current contour can still lose badly on late flip / chop windows. This file is the working checklist so the critical fixes do not get lost while editing.

Primary diagnosis from the code review:

- `j_sell_rescue` can be generated by the strategy, but some runtime paths only execute BUY signals.
- Strategy J mutates internal exposure state when it emits a signal, before paper/live execution confirms the fill.
- The chop gate snapshots only at endgame open; late chaos can still allow directional adds.
- `abortRescueIfAskAbove` exists in config but is not enforced by the composite rescue planner.
- Several legacy `jEndgame` knobs remain visible even though the active planner is the composite `FinalSeal` path.

## P0 Fixes

- [x] Execute both BUY and SELL J signals from all event-triggered paths: market tick, spot tick, trade print.
- [x] Notify the strategy only after a paper fill succeeds; live later must follow the same confirmed-fill rule.
- [x] Move J tier accounting (`rescue_spent_usd`, `hedge_spent_usd`, clips, `primary_side`) from signal emission to confirmed execution notification.
- [x] Keep `sell_rescue_done` false until a sell is actually executed.
- [x] Add a live chop/chaos breaker: if mid-cross counts become too noisy after endgame open, block new directional composite buys while still allowing sell-rescue / flip-hedge.
- [x] Enforce rescue impossibility / expensive-ask guard so the bot does not chase target profit at 97-99c when the math cannot reach target inside caps.
