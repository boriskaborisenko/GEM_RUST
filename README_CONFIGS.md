# Документация по Настройке Конфигураций и Стратегий — GEM_RUST

Этот справочник содержит подробное описание структуры файлов конфигурации (`config.json`) и примеры параметров для поддерживаемых стратегий в нашем плагин-движке **`stratBox`**.

---

## 1. Общие глобальные настройки (`config.json`)

Независимо от выбранной стратегии, на верхнем уровне конфига задаются общие параметры жизненного цикла сделок:

* **`strategy`** (строка) — имя активного плагина стратегии. На основе этого значения бот загружает нужные модули. Варианты:
  * `"simple_both"` — Стратегия А (симметричная покупка, фиксированный тейк).
  * `"asymmetric_ladder"` — Стратегия Б (асимметричный закуп, двухступенчатая лесенка с временным усыханием).
  * `"dynamic_breakeven"` — Стратегия Ц (асимметричный закуп, первая продажа по цели, вторая по формуле динамического безубытка).
  * `"dynamic_grid"` — Стратегия Д (3-ступенчатый Grid + Dynamic BUY + Time-Decay Crossover Filter).
  * `"dynamic_grid_d1"` — Стратегия D1 (one-leg directional pair-builder).
  * `"dynamic_grid_dx"` — Стратегия Dx (current-window fair-value direction + optional hedge/PTB unwind).
* **`exitBeforeEndSeconds`** (число) — за сколько секунд до конца интервала (экспирации) запустить экстренный выход.
* **`forceCloseAtEnd`** (булево) — `true` (включить принудительное закрытие в конце) или `false` (выключить, дать сделке висеть до конца).

### 1.1 Динамическое управление капиталом (`session`)
Параметры блока `"session"` теперь поддерживают интеллектуальный динамический сайзинг позиции в зависимости от текущего показателя **Equity (Эквити)** портфеля бумажного трейдера:

* **`startingBank`** (число) — стартовый баланс сессии (например, `100.0` долларов).
* **`minWindowBudget`** (число) — минимальный защитный лимит бюджета на одну сделку/окно (например, `30.0` USD). Ниже этого значения бот не зайдет при недостатке кэша.
* **`maxWindowBudget`** (число) — максимальный потолок бюджета на одну сделку/окно (например, `150.0` USD) во избежание чрезмерных рисков при разгоне депозита.
* **`windowBudgetPct`** (число) — процент от текущего Equity, выделяемый на закуп одного окна (например, `10.0` означает 10%). Если Эквити вырастет до $450, бюджет раунда автоматически станет $45.
* **`buyUpUsd` / `buyDownUsd`** (числа) — старые статичные лимиты закупа. Используются как резервные (Fallback) только в том случае, если новые поля динамического бюджетирования отсутствуют в конфиге.

**Важно:** глобальный `cheaperSideRatio` удален из `config.json`. Теперь split задает активная стратегия через `EntrySignal`; Strategy D делает это адаптивно по ATR.

### 1.2 Коридор входа перед стартом (`preStartEntry`)
* **`minSideAsk` / `maxSideAsk`** (числа) — диапазон цен спроса (Ask) YES-контрактов для входа в сделку. **Внимание:** Все стратегии теперь полностью динамически считывают эти пороги из конфига (хардкод удален).
* **Рекомендуемый суженный диапазон:** от **`0.48`** до **`0.52`** (для гарантированно сбалансированного закупа около паритета 50/50).

## 2. Стратегия А: «Simple Both» (Базовая «тупая» стратегия)

### Описание логики:
1. **Вход:** За несколько минут до старта закупаем контракты `UP` и `DOWN` на базе динамического бюджета от Equity. Split теперь приходит от активной стратегии, а порог входа динамически считывается из конфига (`0.48 - 0.52`).
2. **Выход:** Мы полностью игнорируем спот-курсы и страйки. Бот просто следит за бидами. Как только бид `UP` или `DOWN` ушел `>= 0.65$` — мы сливаем его на 100%.

### Шаблон файла `config.json` для Стратегии А:
```json
{
  "strategy": "simple_both",
  "minBtcAtr": 0.0,
  "session": {
    "startingBank": 100,
    "minWindowBudget": 30.0,
    "maxWindowBudget": 150.0,
    "windowBudgetPct": 10.0
  },
  "preStartEntry": {
    "enabled": true,
    "minSecondsBeforeStart": 5,
    "maxSecondsBeforeStart": 120,
    "minSideAsk": 0.48,
    "maxSideAsk": 0.52
  },
  "sellStrategy": {
    "exitBid": 0.65
  },
  "exitBeforeEndSeconds": 25,
  "forceCloseAtEnd": false
}
```

## 3. Стратегия Б: «Asymmetric Ladder» (Асимметричная лесенка с распадом)

### Описание логики:
1. **Вход:** Динамический закуп на базе Equity с перевесом в пользу более дешевой стороны на пре-старте опциона.
2. **Выход по лесенке:** Продажи совершаются пошагово (ступеньками), порциями по **`50%`** от купленного объема стороны.
3. **Временной распад (Theta Decay):** Если `"decayEnabled": true`, по мере приближения к концу раунда тейк-пороги усыхают на **10%** в середине раунда и на **20%** в конце. Это позволяет гарантированно выскочить из сделки с прибылью до того, как маркет-мейкеры расширят спреды!

### Шаблон файла `config.json` для Стратегии Б:
```json
{
  "strategy": "asymmetric_ladder",
  "minBtcAtr": 0.0,
  "session": {
    "startingBank": 100,
    "minWindowBudget": 30.0,
    "maxWindowBudget": 150.0,
    "windowBudgetPct": 10.0
  },
  "preStartEntry": {
    "enabled": true,
    "minSecondsBeforeStart": 5,
    "maxSecondsBeforeStart": 120,
    "minSideAsk": 0.48,
    "maxSideAsk": 0.52
  },
  "asymmetricLadder": {
    "strongSteps": [0.62, 0.72],
    "weakSteps": [0.70, 0.85],
    "decayEnabled": true
  },
  "exitBeforeEndSeconds": 25,
  "forceCloseAtEnd": false
}
```

## 4. Стратегия Ц: «Dynamic Break-Even» (Динамический безубыток)

### Описание логики:
Это самый продвинутый алгоритм, полностью нивелирующий коварство временного распада маркет-мейкеров!

1. **Вход:** Динамический закуп на базе Equity с перевесом в пользу более дешевой стороны на пре-старте опциона.
2. **Первый выход:** Ждем достижения `>= 0.65$` (из `"sellStrategy"`) по любой стороне и сливаем её на 100%. Мы вернули большую часть кэша (`Cash_Returned`).
3. **Второй выход (Динамический безубыток):** Бот ждет, пока спот-цена пересечет уровень открытия (страйк) обратно. В эту секунду бот рассчитывает минимальную безопасную цену продажи для второй стороны:
    $$\text{Min\_Safe\_Price} = \frac{\text{Spent} - \text{Cash\_Returned}}{\text{Remaining\_Shares}} + \text{slippageBuffer}$$
   Если текущий Bid в стакане `>= Min_Safe_Price`, бот моментально продает остатки, фиксируя чистую прибыль. Если нет — бот выжидает глубокого разворота спота, либо оставляет под экстренный 15%-й сейв.

### Шаблон файла `config.json` для Стратегии Ц:
```json
{
  "strategy": "dynamic_breakeven",
  "minBtcAtr": 0.0,
  "session": {
    "startingBank": 100,
    "minWindowBudget": 30.0,
    "maxWindowBudget": 150.0,
    "windowBudgetPct": 10.0
  },
  "preStartEntry": {
    "enabled": true,
    "minSecondsBeforeStart": 5,
    "maxSecondsBeforeStart": 120,
    "minSideAsk": 0.48,
    "maxSideAsk": 0.52
  },
  "sellStrategy": {
    "exitBid": 0.65
  },
  "dynamicBreakeven": {
    "slippageBuffer": 0.02
  },
  "exitBeforeEndSeconds": 25,
  "forceCloseAtEnd": false
}
```

## 5. Стратегия Д: «Dynamic Grid» (3-ступенчатый Grid + Dynamic BUY + Time-Decay Filter)

### Описание логики:
Это высокочастотный арбитражный алгоритм институционального уровня, превращающий колебания рынка в чистую прибыль!

1. **Закуп (Hedge):** Динамический закуп на базе Equity со split, который сама Strategy D выбирает по ATR-режиму.
2. **Сетка фиксации (UP):** 
   - Ступень 1: Продать 40% объема по цене **`0.58$`** (быстрый забор микро-прибыли).
   - Ступень 2: Продать 40% объема по цене **`0.66$`** (основной тейк).
   - Ступень 3: Продать 20% объема по цене **`0.75$`** (жадный остаток на случай выноса).
3. **Модуль усреднения «Dynamic BUY» (DOWN):** Если сильная UP дошла до Ступени 3 (`0.75$`), а время раунда прошло **`<= 60%`**, бот моментально докупает слабую DOWN по цене **`0.15$`** лимитным ордером объемом 50% от первоначального, сбивая среднюю цену входа!
4. **Процентный временной фильтр на пересечении:**
   - **`time_pct < 50.0%`:** моментальный рыночный выход из слабой DOWN для фиксации Win (бид `>= 0.43$`).
   - **`50.0% <= time_pct <= 75.0%`:** выход только в том случае, если сумма уже зафиксированной прибыли от UP + текущая цена DOWN покрывает все первоначальные затраты сделки (`Realized_PnL >= 0`).
   - **`time_pct > 75.0%`:** полная блокировка продаж на страйке, даем слабой стороне "долететь" до конца в надежде на полный разворот.

### Шаблон файла `config.json` для Стратегии Д:
```json
{
  "strategy": "dynamic_grid",
  "minBtcAtr": 0.0,
  "session": {
    "startingBank": 100,
    "minWindowBudget": 30.0,
    "maxWindowBudget": 150.0,
    "windowBudgetPct": 10.0
  },
  "preStartEntry": {
    "enabled": true,
    "minSecondsBeforeStart": 5,
    "maxSecondsBeforeStart": 120,
    "minSideAsk": 0.48,
    "maxSideAsk": 0.52
  },
  "sellStrategy": {
    "exitBid": 0.65
  },
  "exitBeforeEndSeconds": 25,
  "forceCloseAtEnd": false
}
```

## 6. Как переключать стратегии?

Вы можете переключать стратегии (мозги бота) в одно слово прямо в файле `config.json`!

* Для запуска **Стратегии А**: Установите `"strategy": "simple_both"`.
* Для запуска **Стратегии Б**: Установите `"strategy": "asymmetric_ladder"`.
* Для запуска **Стратегии Ц**: Установите `"strategy": "dynamic_breakeven"`.
* Для запуска **Стратегии Д**: Установите `"strategy": "dynamic_grid"`.
* Для запуска **Стратегии D1**: Установите `"strategy": "dynamic_grid_d1"`.
* Для запуска **Стратегии Dx**: Установите `"strategy": "dynamic_grid_dx"`.

После подмены файла запустите бота стандартной командой в консоли:
```bash
cargo run -- BTC 5m
```
Бот мгновенно прочитает файл конфигурации и подгрузит нужный плагин со всеми его индивидуальными математическими формулами и порогами!
